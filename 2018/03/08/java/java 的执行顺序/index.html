java 的执行顺序

[toc]

结论

类的加载:

1）创建类的实例，也就是new一个对象

2）访问某个类或接口的静态变量，或者对该静态变量赋值

3）调用类的静态方法

4）反射（Class.forName(“com.lyj.load”)）

5）初始化一个类的子类（会首先初始化子类的父类）

6）JVM启动时标明的启动类，即文件名和类名相同的那个类

父类优先于子类

同一个类

优先级 :(前面的优先) 同一优先级别 代码先后顺序

  • 1 static 静态块 和 static 静态属性

初次实例化执行 以后不执行

  • 2 非静态块

父类的构造方法 先于子类的非静态块

每次实例化的时候都 — 执行

  • 3 对应的构造方法
